一旦项目不只你一个人，就需要建立一些基本规则，并依次将项目的各部分分配出去。

## 务实的团队

- 维持小而稳定的团队
- 禁止破窗
- 团队的工作不应仅致力于开发新功能，还可能包括：
  - 旧系统的维护
  - 流程的反思与精炼
  - 实验新技术
  - 学习和提升技能
- 做能起作用的事，别赶时髦

## 务实的入门套件

- 版本控制
- 回归测试
- 完全自动化

## 傲慢与偏见

在作品上签名。

保持匿名会滋生粗心、错误、懒惰和糟糕的代码，特别是在大型项目中——很容易把自己看成只是大齿轮上的一个小齿，在无休止的工作汇报中制造蹩脚的借口，而不是写出好的代码。
